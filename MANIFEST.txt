CIBOS / CIBIOS / HIP — FULL sandbox archive (steps 1-5 COMPLETE)
================================================================
The full headline product loop runs on the booted kernel, runtime-verified in
QEMU and built bare-metal-first:
  boot -> persistent CIBOSFS (mount-or-format, survives reboot) -> create-user
  -> login (REAL login crate) -> shell (REAL shell::dispatch in ring 3)
  -> compose EXISTING apps pkg/kv/edit (each reused verbatim via process_command)
  -> `pkg install <name>` from a LOCAL /repo (SHA-256 verified, network-free)
  -> installed file in /apps. App baking is flavor-selectable (--with-apps).

Verified: 317 tests pass / 0 fail; 6 SDK examples build; clippy clean; the
on-kernel app crates (shell, package-manager, kvstore, editor, cibos-app,
cibos-console, cibos-sync, accounts, login) all build for x86_64-unknown-none.

New/changed crates this arc:
  cibos-console (Console + ShellFs/ShellSystem seam), cibos-sync (no_std Mutex),
  cibos-app (SyscallConsole/SyscallSystem), accounts/login (no_std), shell/
  package-manager/kvstore/editor (no_std-capable), kernel-image/apps/{login-rs,
  shell-rs} (thin ring-3 launchers for the REAL shared crates).

Step 5 specifics: persistent root fs; package-manager::install_from_repo (verify-
before-write, FS via closures so process_command stays pure); kernel seeds /repo;
shell `pkg install` wired to syscalls; `--with-apps` flavor flag (app-* features).
