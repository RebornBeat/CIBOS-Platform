CIBOS / CIBIOS / HIP — sandbox archive
=======================================
Full canonical workspace (42 crates) + design notes + fresh firmware ELFs.

This archive: the N3-N5 networking sweep (e1000 + from-scratch cibos-net + NIC
transport), verified end to end, plus the complete remaining-arc / per-arch /
WiFi capture and refreshed top-level docs.

  1. N3 — e1000 driver (kernel-image/src/arch/e1000.rs): real Intel 82540EM
     (vendor 0x8086, device 0x100E). PCI discovery, MMIO BAR (identity-mapped),
     EEPROM MAC, legacy RX/TX descriptor rings, polled send/recv. Same NetDevice
     trait. probe_nic_at_boot tries virtio-net first, then e1000; installs the
     first present. Always-compiled production.

  2. N4 — cibos-net (NEW workspace crate, no_std, ZERO external crates,
     forbid(unsafe_code)): from-scratch Ethernet, ARP (+cache), IPv4 (+internet
     checksum), UDP (+pseudo-header checksum), ICMP echo. 15 unit tests, clippy
     clean. TCP deferred to its own milestone. (Aligned: the workspace uses
     outside crates only for cryptography + foundational primitives; a network
     stack is OS logic, written from scratch.)

  3. N5 — net_stack (kernel-image/src/net_stack.rs): NIC-backed UDP transport
     (udp_send_to / poll_udp) over the stored NIC + cibos-net, with ARP resolve,
     on-link/gateway next-hop, and inbound ARP-for-us + ICMP-echo answering
     (the host is pingable). configure(mac) sets the host IP at boot.

  4. VERIFIED end to end in QEMU (real device + real service):
        virtio-net TX: ARP request sent
        virtio-net RX: ARP reply — gw 10.0.2.2 is at 52:55:0a:00:02:02
        net-stack UDP: DNS query sent to 10.0.2.3:53
        net-stack UDP: DNS reply from 10.0.2.3:53 (52 bytes) — STACK OK
        boot complete
     The DNS round-trip exercises the WHOLE stack (ARP -> Ethernet/IPv4/UDP build
     -> NIC TX -> NIC RX -> parse -> port match) against QEMU's real resolver.

  5. REMAINING-ARC-AND-ARCH-WIFI-PLAN.md (NEW, the master capture):
     - A1 wire the Lattice connect()/Link path to net_stack for REMOTE UDP Links
       (surface/APIs unchanged); A2 from-scratch TCP.
     - PART B: the FULL per-arch build flow with the HONEST current state —
       CIBIOS firmware builds for all 4 arches and cibos-kernel core is arch-
       independent, but ring-3 + ALL drivers + interactive surface + networking
       are x86_64-ONLY today (aarch64 = 38-line serial backend; riscv64 = 34-line
       SBI console; i686 shares x86 paths). The sweep brings each non-x86 arch
       from serial+halt to a full runtime (traps, timer, intr controller, MMU,
       console/input, ring-3) -> per-arch drivers (virtio-mmio) -> platforms ->
       a per-arch verification matrix.
     - PART C: WiFi — a from-scratch cibos-wifi 802.11/EAPOL state machine (vetted
       crypto primitives) behind a Wifi/NetDevice surface, plus the connection
       flow per platform (CLI wifi app, GUI network panel, mobile touch picker,
       server declarative config), all behind the Lattice/Warden with encrypted
       credential storage; staged verification.
