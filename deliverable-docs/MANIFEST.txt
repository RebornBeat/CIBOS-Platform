== Dwell: ARM64 image-header flags bug (real-HW correctness) ==
  The header had flags=0xa (bit3=1 = "place kernel anywhere"), but the kernel is
  position-DEPENDENT (linked at a fixed 0x40080000 with absolute symbol refs). A
  conforming real loader honoring bit3 could relocate us and break. QEMU happened
  to place us correctly, hiding it. FIXED to flags=0x2 (text_offset HONORED, bit3
  clear). Verified header bytes + all 3 boot.

== virtio-blk over virtio-MMIO (aarch64/riscv64 storage) ==
  New kernel-image/src/arch/virtio_blk_mmio.rs: real virtio-MMIO transport +
  device-init handshake + split virtqueue + virtio-blk 3-descriptor request
  protocol, implementing the existing BlockDevice trait (MMIO analogue of x86 ATA).
  - Slot array base discovered from the DTB (virtio_mmio@ nodes), NOT hardcoded;
    the driver walks slots and probes each (DeviceID==2). Slot window mapped Device.
  - DMA COHERENCY VERIFIED not assumed: the barrier-only polled path is correct
    only on a coherent platform, so the driver checks the node's `dma-coherent`
    DTB property and WARNS LOUDLY if absent.
    REAL FINDING: aarch64 virt IS dma-coherent; riscv64 virt is NOT — so on real
    RISC-V hardware cache maintenance is required. QEMU TCG models coherent memory
    regardless, so this was an invisible QEMU-era shortcut WITHOUT the check; now
    it is loud. (RISC-V cache-maintenance path is the documented follow-up.)
  VERIFIED:
   - aarch64 (ARM64 Image, real DTB): virtio-blk online at a discovered slot, 2048
     sectors, LBA 0 read returns the EXACT 0x55AA signature written to disk
     (byte-accurate virtqueue DMA read).
   - riscv64 (OpenSBI DTB): same + the correct dma-coherent WARNING.
   - WRITE path: a temporary write+readback of LBA 1 returned BYTE-IDENTICAL data.
   - No-disk case degrades gracefully; x86 ATA path unaffected (full stack OK).
   - 381 tests pass (+1 dma-coherent detection test vs both real DTBs); clean.

== Contract reconciled (honesty) ==
  The probe runs in bring_up_mmu (owns the FrameAllocator the virtqueue DMA needs),

  Done when a disk was found+read OK, else Skipped("no virtio-blk device present")
  — not a blanket "pending block driver". mount_root_fs notes the driver exists and
  root-fs mount is the next step.
