This increment: resolved the legitimate "static vs dynamic" concern by separating
device addresses into THREE tiers and making the board-specific tier DYNAMIC
(DTB-derived) for ARM and RISC-V — keeping only the irreducible bootstrap + true
architectural constants static.

== The principle (STATIC-VS-DYNAMIC-DEVICE-DISCOVERY.md) ==
  Tier 1 — IRREDUCIBLE BOOTSTRAP (must be static): earlycon UART default + the
    firmware/DTB handoff. Required to read anything; universal; every real OS has
    it. KEEP static.
  Tier 2 — ARCHITECTURAL CONSTANTS (legitimately static): x86 VGA 0xB8000, COM1
    0x3F8 — fixed by the PC/VGA architecture on every compatible machine. KEEP
    static (discovering them is pointless; they cannot vary).
  Tier 3 — BOARD-SPECIFIC (must be dynamic): GIC/PLIC/CLINT/board-UART (DTB), x86
    PCI BARs. Hardcoding these = the QEMU-era shortcut. MADE DYNAMIC this increment.

== Tier-3 made dynamic (ARM + RISC-V) ==
  * cibos-dtb: new device_reg(prefix) -> (base, size) reads a node's reg window.
  * aarch64: GIC discovered via dtb_device_reg(b"intc"); static range now EMPTY.
  * riscv64: PLIC (plic@), CLINT (clint@), UART (serial@) discovered; static range
    now EMPTY.
  * Each has a LABELED bootstrap fallback to the QEMU-virt address used ONLY when
    no DTB is present (QEMU -kernel) — registered the same dynamic way, not a
    primary path. Real hardware with a real DTB uses the discovered values.
  * Discovered windows flow into the existing mmio_registry -> carve -> Device map
    (no cap, loud fail). The address SOURCE is now the platform, not a constant.
  VERIFIED:
   - Real-DTB unit tests (captured QEMU DTBs): intc=0x08000000 (ARM); plic=
     0x0C000000, clint=0x02000000, serial=0x10000000 (RISC-V). Discovery proven on
     real platform data, not just fallback.
   - All 3 boot (aarch64/riscv64 MMU online + boot complete; x86 full stack all
     milestones); aarch64 UART works via dynamic/fallback registration.
   - 380 tests pass (+2 DTB discovery tests); all 3 build clean.

== What remains static — and why it's correct ==
  Tier 1 (bootstrap) + Tier 2 (x86 VGA/COM1 architectural constants) only. The x86
  PCI hole is the remaining Tier-3 static item; the honest next step is to
  enumerate the actual device BARs from PCI config space (we already touch PCI for
  the NIC) and register that window, using the same discover->register->carve
  pattern proven here for ARM/RISC-V.
