== DTB PLATFORM DISCOVERY (this milestone) ==
  - cibos-dtb (NEW from-scratch no_std crate, zero external deps): a flattened-
    device-tree parser extracting RAM base/size + device bases. 4 tests INCLUDING
    one against a REAL captured QEMU virt DTB — proves it parses the real-world
    format (what matters for real hardware).
  - kernel_entry now (handoff_ptr, dtb_ptr); riscv64 boot.s preserves a1 (DTB per
    the RISC-V boot protocol), aarch64 passes x0. synth_handoff reads the REAL RAM
    region from the DTB (dtb_ram_region), falling back to the conventional QEMU-
    virt base/size if none is passed. So RAM layout is DISCOVERED, not hardcoded.
  - Honest limitation: QEMU's `-kernel <ELF>` path doesn't pass a DTB pointer
    (it's 0), so that test path uses the labeled fallback; the MECHANISM is proven
    against real DTB data and engages whenever a real pointer arrives (CIBIOS
    firmware / U-Boot / UEFI on real hardware). See DTB-AND-HARDCODED-VALUES.md.
  - Hardcoded-value inventory (DTB-AND-HARDCODED-VALUES.md): aarch64/riscv64 RAM
    base now DTB-driven; peripheral bases (PL011/GIC/UART/PLIC/CLINT) FLAGGED to
    move to DTB device_base next; x86 VGA/COM1/PCI are architecturally fixed (NOT
    QEMU-specific) and correctly remain constants.

== TESTING GUIDE (TESTING-GUIDE.md, new appendix) ==
  Current per-arch build + boot steps (QEMU and real hardware), and crucially a
  LOG-CAPTURE section: CIBOS output is plain serial text (mirrored to VGA on x86),
  NOT written to a file on the USB yet — so capture the serial stream
  (`screen -L`, `-serial file:`, or `| tee`) and send the .log back; on x86 with
  no serial, photograph the VGA screen. Fault lines (ESR/ELR/FAR, scause/sepc/
  stval) are decoded and printed — include them verbatim.

== BOOT VISUALS — honest status ==
  Today the boot is plain monochrome text, top-to-bottom, on serial + VGA text;
  no color, no boxes, no scrollback. A colored/boxed console (the brand palette
  #BBB092/#2E6F6B/#1E2235/#A17A74) is ACHIEVABLE: serial supports ANSI truecolor +
  box-drawing (exact palette); VGA text supports 16 colors + box-drawing
  (approximate palette); true RGB + image logo + graphical login need a framebuffer
  GUI mode (larger future work). A log buffer (for scrollback + on-disk logs) is a
  known future item.
